Decisions made

These are the current working defaults for the MVP and should only change deliberately.

MVP

Dealer-led marketplace

Model A comes first: dealers post finance opportunities and brokers bid.

Motofi is the intro and comparison layer, not the lender.

Vehicle

CarJam day one

Plate lookup is part of the first MVP, with dealer manual override and support override.

Store CarJam fields except valuation; odometer stays manual entry only.

Billing

Stripe subscriptions

Separate broker and dealer subscription support from day one.

Payment is required before paid actions go live; exact prices are still to be filled in.

Data

Buyer context

Listings include buyer income band and credit band with optional notes.

Do not build a full finance application or credit check into MVP.

Bidding

Semi-sealed bids

Dealers see bids. Brokers do not see competitor rates.

Use loose validation first: required rate, term, and conditions.

Close

Accept then unlock

Dealers see broker/company identity before accepting.

Contact details unlock only after a dealer accepts one broker.

Infrastructure

The stack is chosen around speed, familiarity, and enough backend structure to launch a controlled pilot without overbuilding.

Layer Choice Why this choice
Frontend Vite React + TypeScript Simple app shell for dashboards and forms without Next.js overhead.
Backend Convex Owns listings, bids, subscriptions, contact unlocks, audit events, server actions, and live dashboard data.
Auth Clerk Handles sign-in, sessions, user identity, and role-aware app access.
Hosting Vercel Simple preview and production deploys for the React app.
Payments Stripe Billing Broker/dealer subscription gates, with price IDs configurable until amounts are agreed.
Vehicle data CarJam API Server-side plate lookup through Convex; API key stays out of the browser.
CRM Pipedrive Tracks dealer/broker sales pipeline, onboarding, activities, and relationship notes. Convex remains product source of truth.
Email Resend Transactional emails for bid, award, contact unlock, approval, and support updates.

Launch roadmap

The build path is phased so the first usable pilot can be reached without carrying future marketplace complexity into day one.

Phase 0

Repo and environment setup

Create Vite app, Convex project, Clerk auth, Tailwind, routing, env templates, and Vercel preview deploy.

Phase 1

Accounts, roles, subscriptions

Dealer/broker/support roles, approvals, Stripe subscription sync, paid action gates, and Pipedrive account links.

Phase 2

Dealer listing creation

CarJam lookup, manual/support override, vehicle fields, buyer income/credit band, location, deadline, publish flow.

Phase 3

Broker marketplace and bidding

Live listings, listing detail, structured bid form, one active bid per broker, edit/withdraw, competitor rates hidden.

Phase 4

Award and contact unlock

Dealer bid comparison, accept confirmation, listing close, contact unlock, winning/losing broker states.

Phase 5

Support and pilot hardening

Account approvals, CarJam corrections, audit trail, subscription diagnostics, Pipedrive links, demo data.

Phase 6

Controlled pilot launch

Production services, smoke test with dealer/broker/support users, pilot invites, support checklist.
